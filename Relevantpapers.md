# Relevant Papers for Agentic Ops Research

The following papers have been added to the `Papers for reference` directory to support the research outlined in `Paper-ideas-v3.md`.

## 1. AIOps in the Era of Large Language Models (LLMs)
- **File:** `AIOps_LLM_Era_Survey.pdf`
- **Focus:** A comprehensive survey (2020-2024) of how LLMs are transforming AIOps.
- **Relevance:** Provides a baseline for the transition from traditional AIOps to LLM-driven "Agentic Ops".

## 2. AEMA: Verifiable Evaluation Framework for Agentic LLM Systems
- **File:** `AEMA_Agentic_LLM_Evaluation_Framework.pdf`
- **Focus:** Proposes a framework for auditable, multi-step evaluations of multi-agent workflows.
- **Relevance:** Directly supports the "Governance and Guardrails" and "Human-in-Command" sections of the research proposal.

## 3. Agentic Large Language Models: A Survey
- **File:** `Agentic_LLM_Comprehensive_Survey.pdf`
- **Focus:** Categorizes agentic LLMs into reasoning, acting, and interacting.
- **Relevance:** Provides the theoretical foundation for Multi-Agent Systems (MAS) and autonomous agency in operations.

## 4. "AI Oops": Subverting LLM-driven IT Operations via Telemetry Manipulation
- **File:** `AIOps_Doom_Telemetry_Manipulation.pdf`
- **Focus:** Security analysis showing how adversaries can manipulate telemetry to mislead AIOps agents.
- **Relevance:** Essential for the "Risk Vector" and "State Corruption" mitigation strategies in the paper.

## 5. CoE-Ops: Collaboration of LLM-based Experts for AIOps
- **File:** `CoE_Ops_LLM_Experts_AIOps.pdf`
- **Focus:** A framework for specialized agents collaborating on different AIOps tasks.
- **Relevance:** Supports the "Agent-to-Agent (A2A) Collaboration" use case in incident management.

## 6. LLM-based Policy Generation for Intent-based Management
- **File:** `LLM_Policy_Generation_Intent_Management.pdf`
- **Focus:** Uses LLMs to decompose high-level intents into executable policies for application management.
- **Relevance:** Aligns with the "Signal Plane" to "Cognitive Plane" workflow and intent-based service requests.

## 7. Trustworthy Multi-Turn LLM Agents via Behavioral Guidance
- **File:** `Trustworthy_Multi-Turn_Agents_Behavioral_Guidance.pdf`
- **Focus:** Framework for multi-turn tasks under explicit behavioral guidance.
- **Relevance:** Directly addresses the need for "Deterministic Guardrails" for probabilistic models in production environments.

## 8. SRE-Llama: Fine-Tuned LLM for Site Reliability Engineering
- **File:** `SRE_Llama_Autonomous_SLI_SLO.pdf` (arXiv:2511.08282)
- **Focus:** Automates SLI/SLO generation and monitoring using fine-tuned Llama-3, Blockchain, and Federated Learning.
- **Relevance:** Aligns with the "Problem Management: Codebase as RAG" and "Dynamic Change Enablement" sections of your proposal.

## 9. Autonomous Issue Resolver (AIR): Towards Zero-Touch Code Maintenance
- **File:** `AIR_Zero_Touch_Code_Maintenance.pdf` (arXiv:2512.08492)
- **Focus:** A system for unsupervised problem resolution and autonomous code maintenance.
- **Relevance:** Directly supports the "Zero-Touch Resolution" and "Problem Management" use cases.

## 10. A Centroid Based Framework for Text Classification in ITSM
- **File:** `ITSM_Ticket_Classification_Framework.pdf` (arXiv:2511.20667)
- **Focus:** Efficient categorization of support tickets into hierarchical taxonomies.
- **Relevance:** Supports the "Incident Management: Triage and Solve" section by improving signal detection and triage.

## 11. Hierarchical Game-Theoretic Decision-Making for Cooperative Multi-Agent Systems
- **File:** `Hierarchical_Game_Theoretic_MAS_Decisions.pdf` (arXiv:2303.16641)
- **Focus:** Decomposes high-level strategies into executable low-level actions for MAS.
- **Relevance:** Provides a model for "Layer 2: The Cognitive Plane" where agents reason through complex workflows.

## 12. Zero-Touch Reliability: The Next Generation of Self-Healing Systems (Google Scholar)
- **Source:** International Journal of AI, 2024
- **Focus:** Autonomous operations and self-healing systems that minimize human intervention.
- **Relevance:** Theoretical support for the "Observe, Orient, Decide, Act" (OODA) loop execution by MAS.
